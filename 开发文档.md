## 技术栈（最终方案）

| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构/内容（3 个公开页面 + 登录页 + 2 个仪表盘） |
| CSS3 | 视觉样式（Flexbox 布局、响应式适配、深色模式） |
| JavaScript (ES5) | 交互逻辑（灯箱、深色模式、回顶、CloudBase 认证） |
| CloudBase (腾讯云) | 数据库 + 认证 + 云函数（dbProxy） |
| localStorage | 本地数据缓存 + 离线回退 |
| Git + GitHub | 版本控制 + 部署 |

**当前采用 CloudBase（腾讯云）+ localStorage 混合方案**：数据优先从 CloudBase 同步，本地保留缓存以确保离线可用。

## 阶段 5：登录系统 ✅（三次迭代）

### 5.1 Firebase 方案（放弃）
- 谷歌服务在国内被墙，手机无法登录
- 老师白名单机制（硬编码邮箱列表）

### 5.2 LeanCloud 方案（放弃）
- LeanCloud 控制台注册被拒

### 5.3 纯前端方案 ✅（最终采用）
- 零外部依赖，纯 JS + localStorage
- 老师账号在 `auth.js` 硬编码，家长账号存 localStorage
- 支持多老师（`teachers` 数组）
- 角色验证：注册时设 `role: 'parent'`，登录时校验

**当前老师列表：**

**认证 API（`auth.js`）：**
| 方法 | 用途 |
|------|------|
| `Auth.login(email, password)` | 登录，返回用户对象或抛错误 |
| `Auth.register(name, email, password, childName)` | 家长注册 |
| `Auth.logout()` | 登出，清除 session |
| `Auth.currentUser()` | 获取当前登录用户或 null |

## 日常操作指南

### 添加新老师
编辑 `js/auth.js`，在 `teachers` 数组添加：
```js
{ email: '新老师@qq.com', password: '密码', name: '某老师' },
```
然后 `git add -A && git commit -m "添加某老师" && git push`

### 修改老师密码
编辑 `js/auth.js`，改对应老师的 `password` 字段，推送。

### 替换占位图片
1. 把真实照片放到 `images/` 文件夹
2. 在 HTML 中把 `https://placehold.co/...` 替换为 `images/照片名.jpg`
3. 推送

### 本地预览
```bash
cd "G:/春晓画室"
python -m http.server 8080
# 浏览器打开 http://localhost:8080
```

### 部署到线上
```bash
git add -A && git commit -m "描述改了什么" && git push
# 等 1 分钟自动生效
```

## 已知限制 & 未来方向

| 限制 | 原因 | 未来方案 |
|------|------|----------|
| 家长数据存浏览器 | 无后端 | 换后端时迁到数据库 |
| 换设备家长数据丢失 | localStorage 本地 | 接入云数据库 |
| 密码明文存代码 | 纯前端 | 后端加密 |
| 老师无法自行注册 | 设计如此 | 保持，防冒充 |
| 占位图片未替换 | 无素材 | 待填充 |

### 🔌 升级路径
以后店大了想换后端（如腾讯云 CloudBase），只需替换 `js/auth.js` 一个文件，其他页面完全不用改。API 设计保持兼容：
```js
Auth.login(email, password)  → 改内部实现 → 外部不变
Auth.register(...)           → 改内部实现 → 外部不变
```
