## 功能总览

### 对外展示
- 首页：画室简介、6门课程卡片、学员作品、联系信息
- 画廊：美术/书法作品展示 + 课堂照片
- 课程：6门课详细介绍、时间表、报名流程、小班制说明
- 预约：表单提交 → localStorage + ServerChan 微信通知 + 交通指引

### 登录系统
- **家长登录**：邮箱密码 或 邮箱验证码（注册即登录）
- **家长注册**：5 字段（姓名+邮箱+验证码+手机号+孩子名），验证码发到邮箱
- **教师登录**：邮箱密码（2 个预置账号，不可自行注册）
- **忘记密码**：家长可自行重置（CloudBase 邮件），教师需联系管理员
- **密码可见切换** 👁️ / **记住我**（7 天免登录）/ **实时校验** / **按钮 loading 防重复**

## 数据架构

### 存储策略
```
localStorage（主） → 读写即时
CloudBase（备份）  → 异步推送

```

| 集合 | 用途 |
|------|------|

## 认证体系

### 教师账号
在 `js/app/auth.js` → `AUTH_CONFIG.teachers` 硬编码：
| 姓名 | 邮箱 | 角色 |
|------|------|------|
| 张校长 | 756924037@qq.com | admin |
| 郑校长 | 953034984@qq.com | teacher |

### 家长认证
- **邮箱密码**：CloudBase 邮箱注册/登录
- **邮箱验证码**：云函数发邮件 → 验证 → 创建账号 → 自动登录
- 注册字段：姓名、邮箱、验证码、手机号（选填）、孩子姓名
- 新用户注册后自动创建会话，直接进入家长端

### 权限
```js
hasAdminPermission() → user.role === 'admin' || user.role === 'teacher'
```
- admin：张校长，可删除任何记录、CSV 导出、清空预约
- teacher：郑校长，日常操作权限
- 权限覆盖：学生删除、班级删除、记录删除、手动消课等

### 会话管理
- `记住我` 勾选 → localStorage（7 天持久化）
- 不勾选 → sessionStorage（关闭浏览器自动清除）

### 验证码邮件配置

**架构（2026-06-17 更新）：**
- 前端不再通过 SDK `callFunction` 调用（浏览器存在兼容问题），改为 `fetch()` HTTP 直连
- CloudBase HTTP 访问服务 → Event 函数，公开 URL 无需登录态
- 前端 `js/login.js` → `callVerifyCode()` 函数封装

**首次配置：**
1. QQ 邮箱 → 设置 → 账户 → POP3/SMTP → 开启 → 生成授权码
2. CloudBase 控制台 → 云函数 verify-code → 环境变量：
   - `EMAIL_USER` = QQ 邮箱地址
   - `EMAIL_PASS` = 授权码（非 QQ 密码）
3. `email_codes` 集合由云函数自动创建（不再需要手动建表）

---

### 验证码故障排查
```bash
# 1. 检查云函数环境变量
npx tcb fn invoke verify-code --params '{"action":"ping"}'
# → 应返回 hasEmailUser:true, hasEmailPass:true

# 2. 查看最近调用日志
npx tcb fn log verify-code --limit 20

# 3. 测试发送（CLI 调用，绕过前端）
npx tcb fn invoke verify-code --params '{"action":"send","email":"你的邮箱@qq.com"}'

# 4. 常见错误：
# "邮箱授权码无效" → QQ 授权码过期，去 QQ 邮箱重新生成
# "邮件服务器连接失败" → SMTP 网络不通，稍后重试
# "发送过于频繁" → 触发限流，等一会再试
# "email_codes collection not exist" → 云函数自动创建失败
```
