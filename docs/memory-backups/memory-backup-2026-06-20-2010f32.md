# 记忆备份 2026-06-20 (2010f32)

> 本次会话：数据安全三层防护体系

## 问题背景

网站数据分布在 localStorage（浏览器）和 CloudBase（云端），但没有自动化备份机制。历史事故：6/19 localStorage 和 CloudBase 学生数据同时丢失，仅靠一份手动 CSV 恢复。

核心风险：
- localStorage 清空 → CloudBase 能恢复，但课程设置（courses）会丢失
- CloudBase 丢失 → 家长账号永久丢失（无本地镜像）
- 两者同时丢失 → 全毁，仅剩手动下载的 CSV
- JSON 解析异常 → 数据雪崩（损坏→返回空数组→空数组覆盖云端→永久丢失）

## 本次改动

### 🛡️ 第一层：代码级防护

#### safeParseJSON — JSON 异常降级（data.js）
- 新增 `safeParseJSON(key, fallback)` 函数
- 解析失败时：备份损坏原始数据到 `_corrupted_时间戳` key → 返回空数组 → 弹出 SyncBubble 错误提示
- 所有 10 个 `getXxx()` / `saveXxx()` 函数改用安全解析
- 覆盖范围：data.js（10集合）+ parent.js（announcements）+ contact.js（inquiries）
- **防止**：JSON 损坏 → 返回 `[]` → `saveXxx([])` 覆盖云端 → 永久丢失的链式雪崩

#### courses 加入 CloudBase 同步（config.js + data.js + courses.js + core.js）
- `chunxiao-courses` 新增 `_synced` 时间戳 + CloudBase push/pull
- `DEFAULT_COURSES` 移至 data.js 作为兜底值
- courses.js 删除重复的 `getCourses`/`saveCourses`/`DEFAULT_COURSES` 定义
- **关键时序修复**：默认值初始化从 `loadCourses()`（同步前）移到 `pullFromCloud().then()`（同步后），防止刚写入的默认值用最新时间戳覆盖云端已有的自定义课程

#### 数据管理页（data-mgmt.js + teacher-dashboard.html + core.js + dashboard.css）
- 教师端侧边栏新增 💾 数据管理
- 功能：数据总览（10集合统计 + 同步状态）、一键备份（JSON下载）、一键恢复（上传JSON）、单集合导出（JSON/CSV）、清空集合（双重确认 + 同步清空云端）
- DATA_COLLECTIONS 元数据包含 localStorage key → CloudBase 集合名映射

#### 其他修复
- **config.js**: dbProxyUrl 路径从 `G:/Ruanjian/Git/dbProxy` 修正为 `/dbProxy`
- **sync-status.js**: collectionMap 补充 `renewals`（续费）和 `courses`（课程设置）中文标签
- **contact.js**: JSON 解析加 try-catch + 损坏备份
- **parent.js**: announcements 读取改用 `safeParseJSON`

### 🛡️ 第二层：GitHub Actions 每日自动备份

#### backup.cjs（scripts/backup.cjs）
- Node.js CommonJS 脚本，零外部依赖（仅需 `@cloudbase/js-sdk`）
- 注入浏览器 API polyfill：localStorage、XMLHttpRequest、btoa/atob、window
- 匿名登录 CloudBase → 遍历 10 个集合 → 提取 `_type: '_sync'` 文档中的 items
- 输出 `backups/data-YYYY-MM-DD.json` + `backups/latest.json`
- 自动清理 >30 天旧备份
- 集合不存在时优雅降级（视为空集合，不报错）
- 备份 key 使用 localStorage 格式（`chunxiao-students`），与浏览器端 `exportAllData` 完全兼容

#### backup.yml（.github/workflows/backup.yml）
- 定时触发：每天 19:19 UTC（03:19 北京时间）
- 手动触发：workflow_dispatch
- 流程：checkout → 安装 Node 18 → npm install @cloudbase/js-sdk → 运行备份 → git commit + push
- commit 消息含 `[skip ci]` 避免触发额外 CI

## 备份格式兼容性

浏览器导出和 GitHub Actions 导出使用**完全相同**的 JSON 格式：
```json
{
  "version": "2.0",
  "exportedAt": "2026-06-20T...",
  "exportedBy": "...",
  "collections": {
    "chunxiao-students": [...],  // localStorage key
    ...
  },
  "syncedTimestamps": {
    "chunxiao-students_synced": "..."
  }
}
```
两个来源的备份均可通过浏览器端「恢复数据」上传还原。
